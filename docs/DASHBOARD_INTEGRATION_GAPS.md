## Detections-Owned Missing Features

| ID | Gap | Why it matters for the dashboard |
|----|-----|----------------------------------|
| `DET-MISS-001` | Generate a machine-readable dashboard/widget inventory from `scripts/deploy_dashboard.py` | The dashboard needs dashboard names, widget titles, query files, visualization types, and prompts without parsing Python. |
| `DET-MISS-002` | Publish a versioned DTO/schema contract for catalog, dashboard inventory, saved searches, datasets, and log-source fields | The React app needs stable TypeScript types and runtime validation boundaries. |
| `DET-MISS-003` | Export small UI-safe fixture rows for key widgets | Frontend development and tests should not require live OCI access or large ignored `test_data/` files. |
| `DET-MISS-004` | Provide a stable API or MCP read boundary for catalog and query metadata | The dashboard administration page already has an MCP/API shell, but no real content endpoint. |
| `DET-MISS-005` | Add smoke tests for every dashboard widget reference | Broken query paths or visualization metadata should fail before the dashboard consumes generated inventory. |
| `DET-MISS-006` | Publish a log-source field data dictionary | The dashboard log-source explorer needs display names, raw JSON keys, parser ownership, and example values. |

## Export Shape Needed by LoganSecurityDashboardv0

The next artifact should be generated, not hand-maintained:

```json
{
  "version": "1.0",
  "generated_at": "2026-04-28T00:00:00Z",
  "dashboards": [
    {
      "name": "SOC: Browser Attack Detection Dashboard",
      "description": "Browser-side attack detection using SOC Application Logs.",
      "widgets": [
        {
          "title": "APM: Total Browser Attacks (24h)",
          "query_file": "apps/apm_total_attacks_kpi.json",
          "visualization_type": "tile",
          "level": "informational",
          "tags": ["apm", "kpi", "showcase"]
        }
      ]
    }
  ]
}
```

This export should sit beside the catalog, for example `queries/dashboard_inventory.json`, and be regenerated by the same release workflow that updates `queries/catalog.json` and `queries/manifest.json`.

## Integration Rules

- `oci-log-analytics-detections` owns rule authoring, OCL query metadata, log-source parsers, generated inventory, demo datasets, and OCI dashboard deployment scripts.
- `LoganSecurityDashboardv0` owns React routes, UI state, component behavior, visual layout, and frontend tests.
- Cross-project references should use `DASH-MISS-*` for dashboard-owned UI gaps and `DET-MISS-*` for detections-owned export/API gaps.
- Any new dashboard route should point back to a generated artifact in this repository before it introduces static data.
