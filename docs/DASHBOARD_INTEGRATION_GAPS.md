## Export Shape Published for LoganSecurityDashboardv0

`queries/dashboard_inventory.json` is generated from `scripts/deploy_dashboard.py:DASHBOARDS`:

```json
{
  "version": "1.0",
  "generated_at": "2026-04-28T00:00:00Z",
  "dashboards": [
    {
      "name": "SOC: Browser Attack Detection Dashboard",
      "description": "Browser-side attack detection using SOC Application Logs.",
      "widgets": [
        {
          "title": "APM: Total Browser Attacks (24h)",
          "query_file": "apps/apm_total_attacks_kpi.json",
          "visualization_type": "tile",
          "time_selection": {"timePeriod": "l24h"},
          "level": "informational",
          "tags": ["apm", "kpi", "showcase"]
        }
      ]
    }
  ]
}
```

This export should sit beside the catalog, for example `queries/dashboard_inventory.json`, and be regenerated by the same release workflow that updates `queries/catalog.json` and `queries/manifest.json`.

The full artifact contract is documented in `docs/INTEGRATION_SCHEMA.md`.

## Integration Rules

- `oci-log-analytics-detections` owns rule authoring, OCL query metadata, log-source parsers, generated inventory, demo datasets, and OCI dashboard deployment scripts.
- `LoganSecurityDashboardv0` owns React routes, UI state, component behavior, visual layout, and frontend tests.
- Cross-project references should use `DASH-MISS-*` for dashboard-owned UI gaps and `DET-MISS-*` for detections-owned export/API gaps.
- Any new dashboard route should point back to a generated artifact in this repository before it introduces static data.
